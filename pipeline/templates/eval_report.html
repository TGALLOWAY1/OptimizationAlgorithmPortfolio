Validation Methodology

  • Schema Validation — All artifacts are validated against strict JSON schemas ensuring required fields, correct types, and structural integrity.
  • Static Checks — Deterministic analysis detects placeholders (TODO/TBD), unbalanced LaTeX delimiters, duplicated paragraphs, and content quality thresholds.
  • Code Execution — Python implementation examples are executed in isolated subprocesses with timeout enforcement and dependency checking.
  • LLM Judge Evaluation — A high-reasoning LLM evaluates artifacts against rubric criteria (factual accuracy, mathematical correctness, clarity, code quality) grounded in canonical reference facts.
  • Iterative Improvement — Artifacts that fail evaluation are automatically revised using judge feedback, with up to 3 total attempts per artifact.
